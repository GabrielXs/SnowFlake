# SnowFlake — Kotlin Snowflake ID Generator

A Kotlin library that implements the [Snowflake ID](https://en.wikipedia.org/wiki/Snowflake_ID) algorithm for generating unique, time-ordered 64-bit identifiers in distributed systems — without coordination between nodes and without UUID collision risk.

## The problem

UUIDs are random. In high-throughput distributed systems, two independent nodes can generate the same UUID, and random IDs cause index fragmentation in databases. Snowflake IDs solve both problems: they are **globally unique**, **sortable by creation time**, and can be generated by thousands of nodes per millisecond with no central coordinator.

## How it works

Each generated ID is a 64-bit integer composed of three parts:

```
| 41 bits (timestamp) | 10 bits (machine ID) | 12 bits (sequence) |
```

| Part | Bits | Description |
|---|---|---|
| Timestamp | 41 | Milliseconds since epoch — gives ~69 years of range |
| Machine ID | 10 | Identifies the application/node — supports up to 1023 nodes |
| Sequence | 12 | Counter per millisecond per node — up to 4095 IDs/ms/node |

**Maximum throughput with default structure: ~4.1 million unique IDs per millisecond** across all nodes.

The structure is configurable — you can redistribute the 63 bits between timestamp, machine ID, and sequence to fit your use case.

## Installation

Add JitPack to your repositories and the dependency to your module:

```kotlin
// settings.gradle.kts
dependencyResolutionManagement {
    repositories {
        maven("https://jitpack.io")
    }
}

// build.gradle.kts
dependencies {
    implementation("com.github.GabrielXs.SnowFlake:snowflake-generator:main-SNAPSHOT")
}
```

## Usage

### Basic — generate an ID

```kotlin
import dev.gabriel.snowflake.SnowflakeGenerator

val id: Long = SnowflakeGenerator.nextId(applicationId = 1)
println(id) // e.g. 1758430942577664001
```

`applicationId` identifies the node/instance generating the ID. Keep it unique per running instance (0–1022 with the default structure).

### Extract components from an existing ID

```kotlin
val (timestamp, machineId, sequence) = SnowflakeGenerator.extractData(id)

println("Created at : $timestamp ms")
println("Machine ID : $machineId")
println("Sequence   : $sequence")
```

### Custom structure

Redistribute the 63 bits when the default doesn't fit your scale:

```kotlin
import dev.gabriel.snowflake.config.Structure

// Fewer nodes (5 bits = 31 max), more IDs per ms (16 bits = 65535/ms/node)
val structure = Structure.customStructure(
    timeStampBits = 42,
    machineIdBits = 5,
    sequenceBits  = 16
)

val id = SnowflakeGenerator.nextId(applicationId = 1, structure = structure)
```

Constraints: `timestampBits + machineIdBits + sequenceBits` must equal 63.

### Thread safety

`nextId` is `@Synchronized`. Concurrent calls from multiple threads on the same instance are safe — the sequence counter and last-timestamp state are protected.

## Running the tests

```bash
./gradlew test
```

The test suite covers:
- Uniqueness across 4000 sequential calls
- Sequence overflow handling (waits for next millisecond)
- Timestamp and machine ID extraction round-trip
- `applicationId` out-of-range validation
